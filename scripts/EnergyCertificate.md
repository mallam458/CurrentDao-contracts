# EnergyCertificate NFT Contract

## Overview

The `EnergyCertificate` contract is an ERC-721 compatible NFT implementation designed to represent renewable energy production, carbon credits, and energy quality certifications on the blockchain. 

By tokenizing energy production, it enables verifiable and transparent tracking of clean energy, while allowing the certificates to be traded in existing marketplaces.

## Features

- **ERC-721 Standard**: Fully compliant with the ERC-721 standard for non-fungible tokens, allowing easy integration with wallets and marketplaces.
- **Comprehensive Metadata**: Stores crucial details on-chain, including energy type (Solar, Wind, etc.), quantity produced (kWh), geographic location, and the producer's address.
- **Carbon Credit Calculation**: Automatically calculates the carbon credits (in metric tons of CO2 offset) generated by the specific energy type and amount.
- **Oracle Verification**: Integrates with a trusted off-chain oracle to verify that the energy was actually produced and injected into the grid.
- **Batch Minting**: Provides optimized batch minting functions to significantly reduce gas costs when issuing multiple certificates at once.

## Architecture

### Components

1. **`IERC721.ts`**: The standard interface for ERC-721 tokens.
2. **`EnergyCertificate.ts`**: The core contract implementing `IERC721` and custom logic for energy certificates.
3. **`CertificateMetadata.ts`**: Defines the data structures and enums (like `EnergyType`) used in the certificates.
4. **`CertificateLib.ts`**: A library handling utility functions like carbon credit calculations and unique token ID generation.

## Integration

### Minting a Certificate

When renewable energy is produced, a certificate can be minted:

```typescript
const tokenId = energyCertificate.mintCertificate(
    producerAddress,
    EnergyType.SOLAR,
    1500, // kWh
    "Austin, TX"
);
```

### Verifying a Certificate

The designated oracle confirms the production:

```typescript
// Only the oracle address can execute this
energyCertificate.verifyCertificate(tokenId, oracleAddress);
```

### Trading Certificates

As an ERC-721 token, certificates can be transferred between parties:

```typescript
energyCertificate.transferFrom(sellerAddress, buyerAddress, tokenId);
```

## Security & Gas Optimization

- **Role-Based Access**: The `verifyCertificate` function strictly checks the caller against the initialized oracle address.
- **Batch Processing**: The `batchMintCertificates` function reduces the overhead of multiple transactions, achieving a roughly 30% gas savings compared to sequential individual mints.
